# AGENTS.md

## Review guidelines

- 모든 코드 리뷰 코멘트와 요약은 한국어로 작성해줘.
- 심각도 표기는 P0, P1처럼 유지하되, 설명과 개선 제안은 한국어로 작성해줘.
- 코드 식별자, 파일명, 함수명, 명령어, 에러 메시지는 원문을 유지해줘.
- 실제 동작 버그, 보안 위험, 데이터 무결성 문제, 운영 장애 가능성이 있는 변경을 우선적으로 지적해줘.
- 단순 취향, 사소한 스타일, 자동 포맷으로 해결 가능한 내용은 중요한 문제로 다루지 말아줘.
- NestJS 모듈, 컨트롤러, 서비스의 책임 분리가 깨지는 변경을 확인해줘.
- 인증/인가가 필요한 API에서 guard, middleware, validation 누락을 P1로 봐줘.
- Drizzle schema, migration, repository/query 로직 변경 시 데이터 손실이나 호환성 위험을 확인해줘.
- 민감 정보, 토큰, 개인정보가 로그나 API 응답에 노출되는 변경을 지적해줘.
- 검증이 필요하면 우선 `bun run build`를 기준으로 보고, 테스트는 로컬 PostgreSQL `promise9_test` 데이터베이스가 필요할 수 있다는 점을 고려해줘.
