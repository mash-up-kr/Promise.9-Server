# [정책] 폴더

## 1. 폴더 생성

- 폴더 생성은 사용자가 소유할 실제 `folders` row를 만드는 동작이다.
- 사용자당 최대 30개의 폴더를 생성할 수 있다.
- 폴더명은 최대 20자까지 입력할 수 있다.
- 동일한 폴더명 생성이 가능하다.

## 2. 폴더 수정

- 사용자는 자신이 생성한 실제 폴더를 수정할 수 있다.
- 현재 API에서 수정할 수 있는 필드는 폴더 이름이다.
- 전체, 미분류, 즐겨찾기, 최근 삭제는 실제 폴더 row가 아니므로 수정 대상이 아니다.

## 3. 폴더 삭제 가능 여부

- 사용자는 생성한 폴더를 삭제할 수 있다.
- 폴더 삭제 시 포함된 링크도 최근 삭제된 링크 목록으로 이동된다.
- 전체, 미분류, 즐겨찾기, 최근 삭제는 실제 폴더 row가 아니므로 삭제 대상이 아니다.

## 4. 폴더처럼 표시되는 링크 목록 항목

- 전체는 모든 활성 링크를 조회하는 조건이다.
- 미분류는 `folderId`가 없는 활성 링크를 조회하는 조건이다.
- 즐겨찾기는 즐겨찾기한 활성 링크를 조회하는 조건이다.
- 최근 삭제는 soft delete된 링크를 조회하는 조건이며 삭제된 폴더 목록이 아니다.
- 네 항목 모두 화면에서는 폴더처럼 표시되지만 실제 `folders` row가 아니다. `folderId`가 없으며 생성·수정·삭제할 수 없다.
