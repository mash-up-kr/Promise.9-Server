# Error Code Policy

`errorCode`는 `number`로 관리한다.

형식은 `[9][도메인(2자리)][순번(3자리)]`이다.

- `9`: HTTP status code와 구분하기 위한 비즈니스 에러 식별자
- `도메인`: 기능 영역 구분
- `순번`: 도메인 안에서의 에러 순번, `001`부터 시작

## 도메인 코드

| 도메인 | 코드 | 범위 |
|---|---:|---:|
| `COMMON` | `10` | `910001` ~ `910999` |
| `FOLDER` | `20` | `920001` ~ `920999` |
| `LINK` | `30` | `930001` ~ `930999` |
| `IMAGE` | `40` | `940001` ~ `940999` |

## 현재 코드

| errorCode | 의미 |
|---:|---|
| `910001` | 공통 검증 실패 |
| `920001` | 폴더 조회 실패 |
| `920002` | 중복된 폴더 이름 |
| `930001` | 링크 조회 실패 |
| `930002` | 삭제되지 않은 링크 복구 실패 |
| `930003` | 이미 존재하는 링크 |
| `940001` | 이미지 색상 추출 실패 |
| `940002` | 이미지 요청 실패 |
| `940003` | 이미지 요청 시간 초과 |
| `940004` | 이미지 크기 초과 |
| `940005` | 지원하지 않는 이미지 Content-Type |

## 추가 규칙

- 새 코드는 해당 도메인의 마지막 순번 다음 값을 사용한다.
- 같은 원인으로 발생하는 세부 상황은 동일한 `errorCode`를 사용할 수 있다.
- 서로 다른 클라이언트 처리가 필요하면 별도의 순번을 부여한다.
- `HttpException` 계열의 공통 fallback은 HTTP status code를 사용한다.
