# Error Code Policy

`errorCode`는 `number`로 관리한다.

형식은 `[도메인(2자리)][에러종류(2자리)][번호(2자리)]`이다.

- `도메인`: 기능 영역 구분
- `에러종류`: 같은 도메인 안에서의 에러 분류
- `번호`: 같은 분류 안에서의 순번, `01`부터 시작

## 코드 규칙

| 도메인 | 코드 | 설명 |
|---|---:|---|
| `COMMON` | `10` | 공통/전역 예외 |
| `FOLDER` | `20` | 폴더 도메인 |
| `LINK` | `30` | 링크 도메인 |
| `IMAGE` | `40` | 이미지 도메인 |

| 에러종류 | 코드 | 설명 |
|---|---:|---|
| `VALIDATION` | `10` | 입력값 검증 실패 |
| `NOT_FOUND` | `10` | 리소스 없음 |
| `CONFLICT` | `20` | 중복/상태 충돌 |

## 현재 코드

| errorCode | 의미 |
|---:|---|
| `101001` | 공통 검증 실패 |
| `201001` | 폴더 조회 실패 |
| `202001` | 중복된 폴더 이름 |
| `301001` | 링크 조회 실패 |
| `302001` | 삭제되지 않은 링크 복구 실패 |
| `302002` | 이미 존재하는 링크 |
| `401001` | 이미지 색상 추출 실패 |
| `402001` | 이미지 요청 실패 |
| `402002` | 이미지 요청 시간 초과 |
| `402003` | 이미지 크기 초과 |
| `402004` | 지원하지 않는 이미지 Content-Type |

## 추가 규칙

- 새 코드는 기존 도메인 코드와 에러종류 코드를 재사용한다.
- 같은 조합에서 예외가 늘어나면 `번호`만 증가시킨다.
- `HttpException` 계열의 공통 fallback은 HTTP status code를 사용한다.
