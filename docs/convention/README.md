# Convention

Promise.9 팀의 GitHub 사용 규칙입니다.

이슈를 만들거나, 브랜치를 파거나, PR을 올리기 전에 필요한 문서를 확인합니다.

## 문서 목록

| 문서 | 내용 | 언제 볼까 |
| --- | --- | --- |
| [Issue](./issue.md) | 이슈 라벨, 템플릿, 제목 형식 | 이슈를 만들 때 |
| [Branch](./branch.md) | 브랜치 정책, 이름 규칙, 작업 흐름 | 브랜치를 만들 때 |
| [PR](./pr.md) | PR 제목, 본문, 공유 기준 | PR을 올릴 때 |
| [Merge](./merge.md) | Merge commit, 머지 전 확인, 충돌 해결 | PR을 병합할 때 |
| [Commit](./commit.md) | 커밋 메시지 타입과 예시 | 커밋 메시지를 쓸 때 |
