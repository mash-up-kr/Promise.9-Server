# 이슈 컨벤션

GitHub Issues는 작업과 논의를 기록하는 공간입니다.

이슈는 **템플릿 + 라벨 체계**로 관리합니다. 유형별 템플릿으로 내용을 작성하고, 라벨로 분류합니다.

## 라벨 체계

| 라벨 | 설명 | 색상 |
| --- | --- | --- |
| `bug` | 버그를 보고합니다 | `#d73a4a` |
| `feature` | 새로운 기능을 제안합니다 | `#0e8a16` |
| `docs` | 문서를 작성하거나 수정합니다 | `#0075ca` |
| `hotfix` | 긴급하게 수정해야 하는 이슈입니다 | `#b60205` |
| `agreement` | 팀이 함께 정할 합의 사항입니다 | `#fbca04` |

## 이슈 템플릿

`.github/ISSUE_TEMPLATE/`에 아래 템플릿을 두고, 이슈 생성 시 목적에 맞는 템플릿을 선택해 작성합니다.

| 템플릿 | 파일 | 제목 접두사 | 용도 |
| --- | --- | --- | --- |
| Bug report | `bug.md` | `[bug]` | 버그 보고 |
| Feature request | `feature.md` | `[feature]` | 기능 제안 |
| Docs | `docs.md` | `[docs]` | 문서 작업 |
| Hotfix | `hotfix.md` | `[hotfix]` | 긴급 수정 |
| 합의 | `agreement.md` | `[agreement]` | 팀이 함께 정할 사항 논의와 기록 |

## 이슈 제목 작성법

이슈 제목은 `[유형] 요약` 형식으로 작성합니다. 템플릿이 접두사를 채워주면, 뒤에 한 줄 요약을 붙입니다.

```text
[bug] 로그인 시 500 에러 발생
[feature] 약속 생성 API 추가
[docs] CI 문서 보강
[hotfix] 결제 콜백 NPE 긴급 수정
[agreement] 머지 전략 확정
```

접두사는 라벨명과 동일하게 소문자로 씁니다. 요약에는 무엇을 다루는지 드러나야 합니다.
