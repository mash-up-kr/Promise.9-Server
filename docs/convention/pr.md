# PR 컨벤션

PR은 변경 내용을 팀에 공유하는 공간입니다. 승인을 받기 위한 절차보다, 어떤 변경이 들어가는지 알리는 데 목적을 둡니다.

## PR 작성 규칙

- 제목은 `[type] 요약` 형식을 권장합니다.
- 리뷰 가능한 단위로 작게 쪼갭니다.
- 올리기 전에 본인이 먼저 diff를 확인합니다.
- 의미 있는 변경은 최소 2명에게 공유합니다.
- 작업 중인 PR은 Draft PR로 생성합니다.

```text
[feature] 로그인 API 구현
[fix] 토큰 만료 처리 수정
[docs] CI 문서 보강
```

## PR 템플릿

아래 템플릿은 `.github/PULL_REQUEST_TEMPLATE.md`에 등록되어 있습니다.

```md
## 📌 작업 내용
| 무엇을, 왜 변경했는지

<br>

## ✅ 변경 사항
- [ ]
- [ ]

<br>

## 📷 스크린샷 (선택)

<br>

## 🔗 관련 이슈
| close #이슈번호

<br>

## 💬 리뷰어에게
| 중점적으로 봐줬으면 하는 부분
```
